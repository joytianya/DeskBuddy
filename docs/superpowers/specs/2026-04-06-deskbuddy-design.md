# DeskBuddy — 设计文档

**日期：** 2026-04-06  
**类型：** Mac 原生桌面宠物应用  
**目标：** 个人项目

---

## 1. 产品定位

混合型桌面宠物：主要是陪伴，内置轻量 AI 对话能力。宠物有情绪、会互动、能感知用户状态，不是纯装饰也不是全功能助手。

---

## 2. 技术栈

| 层 | 技术 |
|----|------|
| 应用框架 | Swift + SwiftUI |
| 宠物渲染 | SpriteKit（像素精灵图，`filteringMode = .nearest`） |
| 窗口 | NSPanel，always-on-top，可拖拽，点击穿透背景 |
| AI 对话 | Claude API / OpenAI API（用户配置 Key） |
| 语音输入 | Apple Speech Framework / Whisper（可选） |
| 语音输出 | AVSpeechSynthesizer（内置），可选 ElevenLabs |
| 数据持久化 | UserDefaults（设置）+ SQLite/CoreData（对话记忆、亲密度） |

---

## 3. 核心模块

### 3.1 PetEngine
- 加载像素精灵图（sprite sheet，16×16 或 32×32 原始尺寸）
- 整数倍缩放渲染，用户可拖拽调整大小（1x–8x）
- 内置多套皮肤（猫、幽灵、机器人等），用户可导入自定义精灵图
- 动画状态机：idle / happy / sleepy / anxious / bored / excited / clingy
- 每套皮肤每个状态约 4–8 帧动画

### 3.2 EmotionEngine
三个维度加权叠加，输出最终情绪状态：

```
finalMood = time(0.3) + intimacy(0.4) + system(0.3)
```

**时间维度：**
- 06–09 懒洋洋，09–12 活跃，12–14 困倦，14–18 平稳，18–22 放松，22–02 安静，02–06 催睡

**互动维度（亲密度 0–100）：**
- 每次对话 +2，每次点击 +1，静止 30min 后 -1/min
- 0–20 陌生，20–50 熟悉，50–80 亲密，80–100 黏人

**系统感知维度：**
- CPU > 80% → 焦虑；CPU < 20% → 无聊
- 内存紧张 → 求救；静止 > 45min → 来戳你；深夜工作 → 催睡

### 3.3 AIBridge
- 维护对话上下文（最近 N 轮，可配置）
- 系统 prompt 注入当前情绪状态，让 AI 回复风格匹配宠物情绪
- 支持 Claude API 和 OpenAI API，用户在设置中填入 Key
- 对话气泡 UI，支持 Markdown 简单渲染

### 3.4 VoiceModule（可选开关）
- 输入：Apple Speech Framework（离线）或 Whisper API（在线，更准）
- 输出：AVSpeechSynthesizer（内置免费）或 ElevenLabs（付费，更自然）
- 用户在设置中开关，默认关闭

---

## 4. 窗口与交互

- **NSPanel** 悬浮窗，always-on-top，透明背景
- 左键拖拽移动位置
- 右键菜单：设置 / 换皮肤 / 退出
- 双击或点击气泡图标打开对话框
- 宠物边缘拖拽调整大小（整数倍缩放）
- 支持多显示器，记住上次位置

---

## 5. 功能分阶段

### MVP（第一阶段）
- [ ] 像素宠物渲染，3 套内置皮肤
- [ ] 情绪系统（三维叠加）
- [ ] 文字对话（接 AI API）
- [ ] 悬浮窗 + 拖拽 + 右键菜单
- [ ] 基础设置（API Key、皮肤选择、缩放）

### 第二阶段
- [ ] 语音输入 / 输出（可选开关）
- [ ] 亲密度持久化 + 对话记忆
- [ ] 更多皮肤（5–8 套）

### 第三阶段
- [ ] 自定义精灵图导入
- [ ] 皮肤商店（本地包管理）
- [ ] 更丰富的情绪动画帧

## 6. 团队规模

### 一人开发（推荐起步方式）

| 模块 | 预估工时 |
|------|---------|
| Swift/SpriteKit 渲染 + 动画状态机 | 3–4 周 |
| 情绪引擎 + 系统感知 | 1–2 周 |
| AI 对话接入 + 气泡 UI | 1–2 周 |
| 语音 STT/TTS | 1 周 |
| 像素美术（3 套皮肤 × 7 状态 × 6 帧） | 3–4 周 |
| 设置面板 + 打包 | 1 周 |
| **MVP 合计** | **约 10–14 周** |

> 瓶颈在像素美术。建议先用 itch.io 开源像素素材跑通技术流程，再做定制美术。

### 2 人小团队（加速方案）

| 角色 | 职责 |
|------|------|
| 1 × Mac 开发 | Swift + SpriteKit + 系统 API + AI 接入 |
| 1 × 像素美术 | 宠物设计、动画帧、多套皮肤 |

MVP 可压缩至 **4–6 周**，全功能 **2–3 个月**。

### 3 人团队（完整配置）

在 2 人基础上加 1 名产品/测试，负责交互设计细节和 QA，适合想打磨到上架 App Store 的阶段。

---

## 7. 关键风险

| 风险 | 缓解方案 |
|------|---------|
| 像素美术工作量大 | 先用开源素材验证，后期再定制 |
| macOS 系统权限（辅助功能、麦克风） | 提前测试权限申请流程，写清楚隐私说明 |
| AI API 费用 | 用户自带 Key，应用本身不承担费用 |
| NSPanel 在某些 macOS 版本行为不一致 | 锁定 macOS 13+ 作为最低支持版本 |
